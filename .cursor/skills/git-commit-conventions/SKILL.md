# Git 提交约定

## 提交类型

- 新增功能、全新能力或用户可见功能：使用 `feat: 改动描述`
- 迭代修改、缺陷修复、异常处理或兼容性调整：使用 `fix: 改动描述`

## 提交信息要求

1. 只使用一个类型前缀：`feat:` 或 `fix:`
2. 冒号后保留一个空格，再写简洁准确的中文改动描述。
3. 使用单行信息，描述改动结果，不写过程、原因分析或测试日志。
4. 不要使用 `chore:`、`refactor:` 等其他前缀替代上述两类约定。

## 判断规则

- 判断标准是改动的主要目的，而不是文件类型。
- 新增接口、页面、组件、配置能力或自动化流程属于 `feat:`
- 修复已有功能、补充边界处理、调整已有行为或解决运行异常属于 `fix:`

示例：

```text
feat: 新增看板任务 Git 提交 skill
fix: 修复流水线重启后的任务恢复
```
