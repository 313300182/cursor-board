# 🗂️ Cursor Board

**把「提需求 → 写代码 → 跑测试 → 提交 → 部署」整条流水线，交给 AI 自动跑。**

你只管在看板上贴一张任务卡，剩下的开发、测试、Git 提交、部署，Cursor Agent 帮你一条龙搞定；
跑歪了随时插一句话纠正，测不过自动打回重改，忙不过来还能定时自动开工。

一个给个人开发者的「AI 研发流水线看板」。

</div>

---

## ✨ 它能帮你做什么

- 🚀 **一句话交付**：贴张任务卡描述需求，AI 自动完成开发 → 测试 → 提交 → 部署。
- 🔁 **测不过自动重来**：测试失败自动打回继续改，直到通过，无需盯着。
- 🧭 **计划先行**：复杂任务先出方案给你批，批了再动手，不乱改代码。
- 💬 **随时插话**：跑的过程中可以补充说明、让它跳过测试、或一键终止。
- 🖼️ **能贴图**：需求和对话都支持粘贴截图，AI 看图干活。
- ⏰ **定时开工**：常驻任务支持定时（cron）或手动一键触发。
- 🗃️ **多项目多目录**：一个看板管多个项目，前后端目录一起改。
- 🔒 **本地私有**：数据存本地 SQLite，密码登录，代码不出门。

## 🚀 快速开始

### 1. 安装并启动

```bash
npm install
npm start
```

启动后浏览器打开 👉 **http://127.0.0.1:3920**

> 首次访问需要登录，**初始密码：`123456`**，登录后可在界面里修改。
> 端口可在 `config.json` 的 `server.port` 修改。

### 2. 后台常驻启动（可选）

想让它一直在后台跑、开机自启？

| 方式 | 说明 |
|------|------|
| `npm run start:bg` | 后台启动，日志写入 `data/server.log`，端口被占用则跳过 |
| 双击 `start-board.vbs` | 隐藏窗口静默后台启动，适合常驻/开机自启（Windows） |

## 📖 使用说明

### 第一步：建项目

进入首页点「新建项目」，填写：

- **项目名称**
- **工作目录**：AI 实际操作的代码目录，可添加多个（如前端 + 后端一起改）。
- **默认模型 / 默认任务类型**（可选）。
- **Git 提交 / 推送开关**：开启后流水线测试通过会自动 commit（可选 push）。
- **部署命令**（可选）：如 `npm run deploy`。

### 第二步：新建任务

在项目看板点「新建任务」：

1. **选任务类型**：新功能、修 Bug、重构、性能优化、写测试、代码 Review、通用任务……
2. **填需求**：越具体越好，可直接粘贴截图。
3. **选模式**：
   - **流水线模式** —— 开发 → 测试 →（Git 提交）→ 待部署，全自动。
   - **Plan 模式** —— 复杂任务，AI 先给方案，你批准后再执行。
4. **选模型**（可选），提交即入队。

### 第三步：盯进度 / 干预

任务会在看板泳道间自动流转：

```
待执行 → 进行中 → 待人工 → 待部署 → 已完成
                    └──────── 异常（可重试）
```

- 点开任务卡可看**实时日志、AI 提示词、结果摘要**（Markdown 渲染，支持表格）。
- 进行中可以：**补充说明**引导方向 · **跳过测试** · **一键终止**。
- AI 提问且你没及时回复 → 挂到「待人工」，稍后重新入队会**接着上次上下文续跑**。

### 流水线各阶段怎么跑的

| 阶段 | 做什么 |
|------|--------|
| 🛠️ 开发 | 按需求实现代码，只开发，不测不部署 |
| 🧪 测试 | 自动识别或按你指定的命令跑测试；**失败自动打回开发**重改 |
| 📦 Git 提交 | 看板自动挑出本任务改动的文件，AI 只写一行 `feat:/fix:` 提交信息，可选自动 push |
| 🚀 部署 | 进入待部署，触发部署命令；部署失败会自动修复并重试一次 |

### 更多玩法

- **迭代**：已完成的任务可继续提需求，原地续跑同一会话、保留多轮对话。
- **重试**：异常任务点重试，会带着失败原因先修复再继续。
- **Ask 对话**：不想建任务？直接和 AI 聊，纯问答、只读不改文件。
- **常驻任务**：把重复性工作设成定时（cron）或手动触发，每次自动生成一张任务卡。
- **任务类型管理**：全局模板可启用/禁用，还能给项目建专属私有模板。

## ⚙️ 配置速览

配置集中在 `config.json`：

| 配置段 | 作用 |
|--------|------|
| `server` | 监听地址与端口 |
| `cursor` | Agent 程序、各阶段超时、可选模型列表与默认模型 |
| `security` | 工作目录白名单、危险命令拦截、自动审批 / 自动接受计划开关 |
| `queue` | 最大并发、最小空闲内存（防 OOM）等 |

其他目录：

- `templates/` — 内置任务类型模板（feature、bugfix、refactor、optimize、test、code-review、iteration、general…）。
- `data/` — 运行时数据（已 gitignore：SQLite 库、登录密码、token、日志、PID）。

## 🏗️ 项目架构

分层清晰，「装配与启动分离」，路由薄、逻辑在 service、读写在 repository：

```
server.js          装配依赖 + 启动监听
 └─ src/app.js     createApp(deps) 返回 Express 应用
     ├─ routes/    路由层：解析入参 → 调 service/repo
     ├─ services   queue 队列调度 · pipeline 流水线 · acp-runner 会话
     │             chat-service 对话 · schedule-scheduler 常驻任务
     │             project-deployer 部署 · workdir-lock 目录互斥 · git-runner
     └─ db.js      仓储层（SQLite）
public/            原生前端（REST + SSE）
```

数据表：`projects` · `tasks` · `task_events` · `chat_sessions` · `chat_messages` · `project_templates` · `schedules`
